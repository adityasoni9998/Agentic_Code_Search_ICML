\begin{abstract}
A prerequisite for resolving issues in large codebases is code localization---the identification of relevant files, classes, and functions in the repository that must be modified.
While localization was originally performed through vector search, recent work has focused on agents themselves performing the localization task, either prior to or jointly with solving the task itself.
Most previously published research studies on agentic code search have focused on providing the agent with extra tools such as repository graphs using static analysis tools, or vector indices.
In this paper, we demonstrate that, with a strong reinforcement learning recipe, a largely out-of-the-box coding agent can be trained to achieve surprisingly strong results, beating all previously reported results for its model size, and sometimes approaching results provided by closed models such as Claude Sonnet using specialized scaffolds.
In particular, we focus on the techniques used to repurpose existing coding agent environments into environments for code search, reward design, and optimization techniques.
We release the resulting model family, \methodname, and all training code for the community to build upon.
