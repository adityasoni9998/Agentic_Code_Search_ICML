\begin{abstract}
A prerequisite for resolving issues in large codebases is code localization - the identification of relevant files, classes, and functions in the repository that must be modified. Effective localization requires navigating complex software repositories and rapidly acquiring contextual understanding. Traditionally, automated software engineering systems rely on a single agent to perform both code localization and code editing. However, recent work in the agent building community has shown that we can train specialized models as highly efficient subagents to perform the code localization subroutine. Despite this promise, most existing approaches are either closed-source or design complex scaffolds that are difficult to use and do not generalize across programming languages. In this work, we present an open-source recipe for training a dedicated code localization agent. We re-purpose established issue-resolution benchmarks (SWE-Smith and SWE-Bench) for localization tasks and integrate them with the OpenHands agent harness.
